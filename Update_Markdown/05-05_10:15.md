# 2026 년 5 월 5 일 - Anthropic 디자인 시스템 구현

## 📋 작업 개요

이 세션에서는 **Anthropic 의 디자인 가이드라인**을 기반으로 대시보드 UI 를 리디자인했습니다.

---

## 🎯 왜 이 작업을 했나요?

### Anthropic 디자인이란?
Anthropic 은 Claude AI 를 만든 회사로, 그들의 디자인은 다음과 같은 특징이 있습니다:
- **따뜻한 색감** - 차가운 파란색 대신 코랄, 크림색 사용
- **서체 조합** - 제목은 Serif(세리프), 본문은 Sans(산세리프)
- **부드러운 대비** - 눈이 피로하지 않은 중간 명도

---

## 🔧 무엇을 변경했나요?

### 1. 색상 팔레트 변경

#### 기존 색상 (표준 파란색 테마)
```javascript
primary: '#2563eb',      // 파란색
bg: '#f8fafc',          // 차가운 회백색
text: '#1e293b'         // 진한 회색
```

#### 새로운 색상 (Anthropic 테마)
```javascript
primary: '#cc785c',      // 코랄 오렌지 🍊
canvas: '#faf9f5',       // 따뜻한 크림색 📄
surface-card: '#efe9de', // 카드 배경
ink: '#141413'           // 깊은 검정 (텍스트)
```

**왜?** 파란색은 너무 흔하고 차가운 느낌입니다. 코랄과 크림색은 따뜻하고 친근한 느낌을 줍니다.

---

### 2. 폰트 변경

#### 기존 폰트
```javascript
sans: ['Inter', 'system-ui', 'sans-serif']
```

#### 새로운 폰트 조합
```javascript
display: ['Cormorant Garamond', 'Georgia', 'serif'],  // 제목용
sans: ['Inter', '-apple-system', 'sans-serif'],        // 본문용
mono: ['JetBrains Mono', 'monospace']                  // 코드용
```

**왜?** 
- **Serif(세리프)** 폰트는 제목에 사용하면 우아하고 전문적인 느낌
- **Sans(산세리프)** 폰트는 본문에 사용하면 읽기 편함
- 두 가지를 조합하면 시각적 계층 구조가 명확해짐

---

### 3. 컴포넌트 클래스 정의

#### 버튼 스타일
```css
/* 프라이머리 버튼 - 주요 행동 */
.btn-primary {
  @apply bg-primary text-on-primary px-5 py-2.5 rounded-md;
}

/* 세컨더리 버튼 - 보조 행동 */
.btn-secondary {
  @apply bg-canvas text-ink px-5 py-2.5 rounded-md border border-hairline;
}
```

#### 카드 스타일
```css
.card {
  @apply bg-surface-card rounded-lg p-8 mb-6;
}
```

**왜?** 같은 스타일을 여러 번 반복해서 쓰지 않고, 클래스 하나로 재사용할 수 있습니다.

## 📁 변경된 파일 상세

### 1. `tailwind.config.js`

**이 파일은 무엇인가요?**
Tailwind CSS 의 설정 파일입니다. 색상, 폰트, 간격 등 디자인 시스템을 정의합니다.

```javascript
module.exports = {
  theme: {
    colors: {
      // 브랜드 색상
      primary: '#cc785c',
      'primary-active': '#a9583e',
      
      // 배경 색상 계층
      canvas: '#faf9f5',       // 전체 배경
      'surface-card': '#efe9de', // 카드 배경
      'surface-soft': '#f5f0e8', // 부드러운 배경
      
      // 텍스트 색상 계층
      ink: '#141413',          // 가장 진한 텍스트
      'body-strong': '#252523', // 강조 텍스트
      body: '#3d3d3a',         // 일반 텍스트
      muted: '#6c6a64',        // 보조 텍스트
    },
    
    fontFamily: {
      display: ['Cormorant Garamond', 'serif'],  // 제목
      sans: ['Inter', 'sans-serif'],              // 본문
      mono: ['JetBrains Mono', 'monospace'],      // 코드
    },
  },
}
```

---

### 2. `src/app/globals.css`

**이 파일은 무엇인가요?**
전역 CSS 파일입니다. Tailwind 의 기본 설정과 커스텀 컴포넌트를 정의합니다.

```css
@import "tailwindcss";

/* 폰트 가져오기 */
@import url('https://fonts.googleapis.com/css2?family=Cormorant+Garamond&family=Inter&display=swap');

/* 기본 스타일 */
@layer base {
  body {
    @apply bg-canvas text-body font-sans;
  }
  
  h1, h2, h3 {
    @apply font-display;  /* 제목에 Serif 폰트 사용 */
  }
}

/* 컴포넌트 정의 */
@layer components {
  .card {
    @apply bg-surface-card rounded-lg p-8;
  }
  
  .btn-primary {
    @apply bg-primary text-white px-5 py-2.5 rounded-md;
  }
}
```

### 3. `src/app/dashboard/page.tsx`

**이 파일은 무엇인가요?**
대시보드 페이지의 메인 컴포넌트입니다.

```tsx
<h1 className="font-display text-4xl font-medium mb-1 text-ink">
  클라우드 대시보드
</h1>

<button className="btn-primary" onClick={() => setShowCreateModal(true)}>
  새 VM 생성
</button>
```

**변경된 점:**
- `text-ink` - 새로운 텍스트 색상 사용
- `font-display` - Serif 폰트로 제목 표시
- `btn-primary` - 커스텀 버튼 클래스 사용

```
┌─────────────────────────────────────┐

│  ┌───────────────────────────────┐  │
│  │  Surface-Card (#efe9de)        │  │
│  │  - 카드, 섹션 배경             │  │
│  │                               │  │
│  │  [Primary Button]              │  │

│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
```

### 2. 텍스트 색상 계층

| 클래스 | 색상 | 사용처 |
|--------|------|--------|
| `text-ink` | #141413 | 중요한 제목 |
| `text-body-strong` | #252523 | 강조된 텍스트 |
| `text-body` | #3d3d3a | 일반 본문 |
| `text-muted` | #6c6a64 | 보조 텍스트, 설명 |

### 3. 타이포그래피 규칙

```
제목 (H1) → Cormorant Garamond, 4xl, font-medium
제목 (H2) → Cormorant Garamond, 2xl, font-medium
본문      → Inter, base, font-normal
캡션      → Inter, xs, uppercase, tracking-wider
```

## 💡 초급자가 알아야 할 것

### Q: `@apply` 는 무엇인가요?

**A:** Tailwind 의 유틸리티 클래스를 CSS 로 묶어주는 기능입니다.

```css
/* ❌ 나쁜 예 - 매번 똑같은 클래스 반복 */
<button class="bg-primary text-white px-5 py-2.5 rounded-md">
<button class="bg-primary text-white px-5 py-2.5 rounded-md">

/* ✅ 좋은 예 - 컴포넌트 클래스 사용 */
<button class="btn-primary">
<button class="btn-primary">

/* 컴포넌트 정의 */
.btn-primary {
  @apply bg-primary text-white px-5 py-2.5 rounded-md;
}
```

### Q: 왜 색상 값을 직접 쓰지 않고 변수를 쓸까요?

**A:** 나중에 색상을 한 번에 변경할 수 있습니다.

```javascript
// ❌ 나쁜 예 - 색상 값이 여기저기 흩어져있음
<button style={{ backgroundColor: '#cc785c' }}>
<div style={{ color: '#cc785c' }}>

// ✅ 좋은 예 - 변수로统一管理
<button class="bg-primary">
<div class="text-primary">

// 나중에 색상을 변경할 때
primary: '#ff6b6b'  // 한 곳만 변경하면 끝!
```

### Q: `font-display` 와 `font-sans`의 차이는?

**A:** 
- `font-display` - 제목에 쓰는 Serif 폰트 (Cormorant Garamond)
- `font-sans` - 본문에 쓰는 Sans 폰트 (Inter)

```tsx
<h1 class="font-display">제목 - 우아한 Serif</h1>
<p class="font-sans">본문 - 깔끔한 Sans</p>
```
