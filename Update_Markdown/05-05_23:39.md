## 2. 해결 방법 (리팩토링 후 구조)

### 새 폴더 구조

```
src/app/dashboard/
  page.tsx                   ← 레이아웃만 담당 (128줄로 축소!)
  types.ts                   ← 타입 정의 (기존 유지)
  api/
    apiClient.ts             ← API 함수 모음 (기존 유지)
  _hooks/
    useDashboard.ts          ← 모든 상태와 액션을 담은 훅
  _components/
    Modal.tsx                ← 재사용 가능한 모달 뼈대
    StatusBadge.tsx          ← VM 상태 표시 배지
    PasswordCell.tsx         ← SSH 비밀번호 표시/숨기기/복사
    QuotaItem.tsx            ← 쿼터 단일 항목 (진행 바)
    QuotaSection.tsx         ← 쿼터 5개 항목 그리드 전체
    VmTable.tsx              ← VM 목록 테이블
    CreateVmModal.tsx        ← VM 생성 폼 모달
    PortForwardModal.tsx     ← 포트포워딩 관리 모달
    SpecChangeModal.tsx      ← vCPU/RAM 사양 변경 모달
    JobProgressModal.tsx     ← VM 생성 진행 상황 모달
```

> **왜 폴더 이름 앞에 `_`(언더스코어)를 붙였나요?**
> Next.js App Router에서 `_`로 시작하는 폴더는 URL 경로로 인식되지 않습니다.
> `_components`, `_hooks` 폴더가 실수로 페이지(route)로 등록되는 것을 막기 위해 사용합니다.

## 3. 각 파일 역할 설명

### `page.tsx` (메인 페이지)

**이 파일만 보면 전체 화면 구조를 파악할 수 있습니다.**

```tsx
// 이런 구조로 단순해졌습니다
export default function DashboardPage() {
  const dash = useDashboard(); // 모든 상태와 액션이 여기에 있음

  return (
    <div>
      <header>...</header>           // 헤더 (이름, 버튼들)
      <QuotaSection />               // 쿼터 사용량
      <VmTable />                    // VM 목록 테이블
      {showCreateModal && <CreateVmModal />}    // VM 생성 모달
      {pfVm && <PortForwardModal />}           // 포트포워딩 모달
      {specVm && <SpecChangeModal />}          // 사양 변경 모달
      {showJobsModal && <JobProgressModal />}  // Job 진행 모달
    </div>
  );
}
```

---

### `_hooks/useDashboard.ts` (상태 관리 훅)

**"무엇을 할 수 있는지"를 모아둔 파일입니다.**

모든 `useState`, `useEffect`, API 호출 함수(VM 생성·삭제·전원제어·포트포워딩 등)가 여기에 있습니다. `page.tsx`는 이 훅을 `const dash = useDashboard()`로 불러와서 사용합니다.

```tsx
// 훅이 반환하는 것들 (일부)
return {
  vms,             // VM 목록 데이터
  quotas,          // 쿼터 데이터
  handleCreateVm,  // VM 생성 실행 함수
  handleDeleteVm,  // VM 삭제 실행 함수
  handleVmAction,  // 전원 시작/정지/재시작 함수
  handleCreatePf,  // 포트포워딩 생성 함수
  // ...
}
```

> **Custom Hook이란?**
> `use`로 시작하는 함수로 React의 상태(`useState`)와 부수효과(`useEffect`)를 묶어서 재사용할 수 있게 만든 것입니다.
> 로직과 화면 코드를 분리하는 가장 기본적인 React 패턴입니다.

---

### `_components/Modal.tsx` (재사용 모달 뼈대)

모달 4개(VM 생성, 포트포워딩, 사양 변경, Job 진행)가 모두 이 컴포넌트를 감싸서 사용합니다. 모달의 배경 오버레이, 흰 카드, 제목, 닫기(×) 버튼이 여기에 정의되어 있습니다.

```tsx
// 사용 예시 — 모달 안에 원하는 내용만 넣으면 됩니다
<Modal title="새 가상 머신 생성" onClose={() => setOpen(false)}>
  <p>여기에 모달 내용을 작성하세요</p>
</Modal>
```

---

### `_components/VmTable.tsx` (VM 테이블)

VM 목록 테이블 전체를 담당합니다. 내부에 `VmTableRow`(단일 행), `SpecLabel`(사양 표시), `Sep`(구분자) 소컴포넌트가 함께 있습니다.

각 행의 버튼(▶ 시작, ■ 정지, ↻ 재시작, 사양, ✕ 삭제)도 이 파일에 있습니다.

## 4. 중복 코드 제거 내용

### 제거된 중복 `authFetch`

**기존 `page.tsx`에 있던 코드 (삭제됨):**
```tsx
// page.tsx 상단에 있었지만 api/apiClient.ts 에 이미 동일한 코드가 있었음
function authFetch(url: string, options: RequestInit = {}) {
  const token = typeof window !== 'undefined' ? localStorage.getItem('access_token') : '';
  return fetch(url, { ...options, headers: { Authorization: `Bearer ${token}` } });
}
```

**변경 후:** `useDashboard.ts`에서 `api/apiClient.ts`의 `authFetch`를 import해서 사용

## 6. 앞으로 새 기능 추가할 때

1. `_components/SnapshotModal.tsx` 파일 새로 생성 (기존 모달 파일 참고)
2. `_hooks/useDashboard.ts`에 스냅샷 상태와 `handleCreateSnapshot()` 함수 추가
3. `page.tsx`에 `<SnapshotModal />` 한 줄 추가
4. `api/apiClient.ts`에 `createSnapshot()` API 함수 추가
5. `types.ts`에 `Snapshot` 타입 추가

각 파일의 역할이 명확해서 어디에 무엇을 추가해야 할지 바로 알 수 있습니다.

## 7. 기술 용어 정리

| 용어 | 설명 |
|------|------|
| **Custom Hook** | `use`로 시작하는 함수. 상태와 로직을 컴포넌트에서 분리해 재사용 가능하게 만드는 React 패턴 |
| **Props** | 부모 컴포넌트에서 자식 컴포넌트로 전달하는 값. 함수의 매개변수와 비슷한 개념 |
| **리팩토링** | 동작은 바꾸지 않고 코드의 구조를 개선하는 작업 |
| **관심사 분리** | 상태 관리, UI 렌더링, API 호출을 각각 다른 파일에서 담당하게 나누는 설계 원칙 |
| **`_` 언더스코어 prefix** | Next.js에서 URL 경로로 등록되지 않는 "비공개" 폴더를 만들 때 사용 |
